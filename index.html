About me...

Hi there! I’m Rachel, a software engineer with two years of hands-on experience in building dynamic applications and solving complex problems. My journey into tech began with a fascination for coding, and it has evolved into a passion for creating user-friendly solutions that make a difference.

I have a strong foundation in Python, Java, and JavaScript, and I enjoy working on both front-end and back-end development. Recently, I developed and implemented a Python REST API using FastAPI, automating control management regulations and scoring processes, which reduced processing time from two months to an average response time of 400 ms—significantly increasing efficiency for internal teams. I also designed and implemented robust backend API components for a banking application using Spring Boot, Java, and PostgreSQL, enhancing data intake from multiple sources and improving overall application performance.

My work includes engineering backend calculations for key performance metrics displayed on dashboards, providing actionable insights across all levels of U.S. Bank, from VP to application level. Additionally, I contributed to front-end development using JavaScript and Angular, resolving minor bugs and implementing dashboard enhancements that improved user experience.

I also automated the data migration process from MUFG Bank to U.S. Bank using custom Python scripts and SQL queries, efficiently transferring multiple terabytes of critical data and significantly enhancing operational efficiency. To ensure code quality and security, I implemented comprehensive unit tests using JUnit and Mockito for both frontend and backend, resulting in a 60% increase in code coverage and reduced bug rates.

Outside of coding, I love to explore North Carolinas beautiful hiking trails with my dog Eli and my cat Beetle Juice, reading, or playing videos games.
